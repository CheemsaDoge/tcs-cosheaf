# 产品规格

## 目的

TCS-Cosheaf 是一个由 Git 支持的类型化知识库和代理工具框架，用于理论计算机科学研究。该产品的存在是为了使研究制品在代码库中变得明确、可审查、可验证和可重复。

## MVP 范围

MVP 包括：

- 由 Git 支持的类型化知识库。
- 模式（schema）验证。
- 制品加载器。
- 依赖图。
- 关卡（gatekeeper）。
- 上下文包生成器。
- 验证器适配器接口。
- Python 检查器适配器。
- 最小化可选的 SAT DIMACS 适配器。
- 最小化可选的 SMT-LIB 适配器。
- 最小化可选的 Lean 纯文件适配器。
- 运用 Python 检查器和可选的 SAT 风格证据路径的小型草稿试验制品。
- GitHub Actions CI。

## 明确不在 MVP 范围内

MVP 不包括：

- 没有 Web UI。
- 没有模型训练。
- 没有自动定理证明代理。
- 没有完整的 Lean 自动形式化。
- 没有多用户权限系统。

## MVP 成功标准

MVP 应该允许贡献者或代理添加类型化制品、验证它们、检查依赖、运行关卡、生成任务上下文，并报告验证器结果，而无需依赖不可重复的本地状态。

当前的试验故意设计得较小且仅限草稿。图论试验演示了在有限图上的本地 Python 检查器证据。SAT/CNF 试验演示了一个微小的可满足公式、一个记录的赋值、在后端可用时可能被跳过或执行的可选 SAT 求解器证据，以及一个 Python 备用检查器。当存在如 `z3` 等可选后端时，SMT 适配器可以执行微小的代码库本地 SMT-LIB 证据，否则将记录为跳过而不是通过。这些路径用于演练工作流基础设施；它们不声称新定理或完整的自动定理证明。

## 非目标

该项目不试图取代同行评审、形式化证明助手或领域专家的判断。它为保存和检查研究状态提供了一个结构化的基础层。
