# v0.12.0 - Research Memory Learning + Benchmark Suite v1

Status: release candidate. Package metadata and `cosheaf.__version__` record
`0.12.0`. The public tag, GitHub release, post-tag release smoke, and
downstream workspace-template/public-KB pin updates are future Phase F.2 work.

This release candidate packages the V17 Research Memory Learning + Benchmark
Suite v1 line on top of the published `v0.11.0` External Operator Campaigns
baseline.

## Included Surfaces

- Deterministic sidecar memory updates under `.cosheaf/memory/`:
  update-run sidecars and aggregate weights.
- Memory CLI:
  `cosheaf memory update-from-workflow`, `update-from-campaign`, `rebuild`,
  and `explain`.
- Benchmark suite v1:
  `cosheaf benchmark list`, `run --suite <suite>`, and `report`.
- Comparative reports:
  `cosheaf compare workflows`, `campaigns`, and `benchmarks`.
- Static review reports:
  `cosheaf report workflow`, `campaign`, and `benchmark`.
- Documentation for memory updates, benchmark runs, comparisons, static
  reports, and V17 authority boundaries.

## Authority Boundary

Memory weights are deterministic sidecar guidance only. Benchmark runs,
comparisons, and static reports are regression or review context only. They are
not proof, source metadata, human review, verifier pass, gate pass, accepted
status, accepted theorem/refutation, or promotion authority.

Skipped, unsupported, unavailable, and inconclusive rows are not passes.

This release candidate does not add model training, automatic correctness
improvement, automatic theorem proving, Lean autoformalization,
CSLib/mathlib proof checking, informal/formal semantic alignment, automatic
accepted promotion, accepted writes, human-review creation, default hosted
provider calls, arbitrary shell execution, web UI, production autonomy, or
multi-user permissions.
