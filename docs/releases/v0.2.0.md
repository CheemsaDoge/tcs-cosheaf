# v0.2.0 - Local MVP Release

`v0.2.0` packages the deterministic local-MVP surfaces that landed after the
`v0.1.1` Formal Link Layer baseline. It is a pin-able framework version for
local three-repository demos and reviewable research workflows.

This release is not a production-readiness claim. It does not add a hosted LLM
runtime, web UI, automatic theorem proving, automatic accepted promotion,
multi-user permissions, or automatic informal/formal semantic alignment.

## Highlights

- Deterministic librarian surfaces:
  artifact cards, lexical and SQLite FTS/BM25 search, memory graph/PageRank,
  Personalized PageRank, score breakdowns, and retrieval audit metadata.
- Context pack v2:
  card-first context packs, explicit full-artifact pull budgets, public-only
  filtering, and conservative formal-link display.
- Local orchestrator dry-run:
  state-machine contracts, task DAG planning, worker-bundle v2 validation,
  local worker execution, reducer records, and inspectable run logs.
- Provider-neutral model interface:
  deterministic fake provider and role contracts that keep hosted execution
  disabled by default.
- Evaluation and observability:
  retrieval evals, context-pack regression evals, structured run logging, and
  optional OpenTelemetry run-log export scaffolding.
- Optional source and formal-tool staging:
  local-file MarkItDown conversion as staging only, plus optional external Lean
  library reference `import` / `#check` checks when Lean or lake is available.

## Boundaries

- Hosted LLM/model-provider execution is not implemented.
- Missing optional tools remain `skipped`, not `pass`.
- A successful Lean `#check` means only that an import and symbol resolved. It
  does not prove informal/formal semantic alignment.
- Formal links remain metadata unless a verifier actually runs and records a
  result.
- Worker output, tool output, retrieval scores, AI review, sidecars, generated
  dashboards, and context packs cannot become accepted knowledge by themselves.
- Accepted public KB artifacts still require source metadata and human review;
  validation and gate success are not substitutes for review.

## Downstream Guidance

Downstream workspaces that need the local-MVP surfaces should pin to:

```text
git+https://github.com/CheemsaDoge/tcs-cosheaf.git@v0.2.0
```

Use `tcs-cosheaf-workspace-template` as the user-facing entry point. Mount or
checkout `tcs-kb-public` as readonly public knowledge and keep private work in
the writable private KB overlay.

Before depending on this tag for a downstream showcase, run the workspace demo
and the public KB validation/gate checks against the tag.
