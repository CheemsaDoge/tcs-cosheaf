# v0.1.1 - Formal Link Layer Support

`v0.1.1` is the Formal Link Layer support release for TCS-Cosheaf. It lets
artifacts record metadata references to external Lean declarations while
keeping formal proof libraries outside the repository.

This note describes the `v0.1.1` tag snapshot. The current `main` branch may
contain later release-hardening work that is not included in the tag.

## Highlights

- Adds artifact metadata fields for formal-link workflows:
  `formalizations`, `alignment`, and `verification_policy`.
- Adds G10 Formal Link Gate for static consistency checks between
  verification policy, formalization references, and alignment review metadata.
- Surfaces formal-link metadata in context packs without claiming Lean
  verification.
- Extends deterministic index output with SQLite `formalizations` and
  `artifact_formal_policy` tables.
- Extends `.cosheaf/artifact_manifest.json` with compact formalization,
  alignment-status, and verification-policy metadata.
- Adds read-only `ArtifactIndexQuery` methods for formalizations and formal
  policy rows.

## Metadata-Only Boundary

Cosheaf does not replace CSLib, mathlib, Lean, or human semantic review. YAML
stores references to formal declarations, not Lean proof bodies.

This release does not implement:

- Lean execution for external formal library references.
- CSLib, mathlib, lake, or Lean dependencies.
- External formal library fetching.
- Automatic informal/formal statement alignment.
- Accepted-promotion policy changes beyond ordinary gatekeeper blocking
  behavior.

Missing optional Lean tooling remains a skipped verifier result, not a pass.
A Lean pass, when future tooling exists, will still not automatically prove that
an informal artifact statement is semantically aligned with a formal
declaration.

## Downstream Guidance

Downstream repositories should pin to `v0.1.1` before using
`formalizations`, `alignment`, or `verification_policy` in artifact YAML.
Formalization links should use `planned` until an external review/checking
workflow has actually established stronger status.
