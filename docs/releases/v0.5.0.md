# v0.5.0 - Operator MCP + Codex Application Layer

Status: release candidate metadata prepared. Package metadata and
`cosheaf.__version__` record `0.5.0` in the release-candidate branch. The
public `v0.5.0` tag, GitHub release, post-tag release smoke, and downstream
workspace/public-KB pin updates are not part of this release-candidate task.

This release candidate packages the Operator MCP + Codex Application Layer
work that landed after the published `v0.4.0` Strategy Planner + Research Task
Graph release. It makes the existing CLI-first research workflow easier for
Codex-style external operators to drive through documented runbooks, optional
MCP tools, and controlled draft/review/runtime write surfaces while preserving
Cosheaf's existing validation, gate, human review, public/private, verifier,
provider, and promotion boundaries.

This release candidate is not production-ready software. It does not add
automatic theorem proving, full Lean/mathlib/CSLib proof checking of informal
statements, automatic autoformalization, automatic informal/formal semantic
alignment, automatic accepted promotion, AI-as-human-review, a web UI, SaaS
behavior, multi-user permissions, default-on provider calls, or MCP authority
over accepted knowledge.

## Highlights

- Read-only operator MCP surface:
  the optional stdio MCP adapter exposes whitelisted workspace, validation,
  gate, PR-checklist, public memory, public-only context, public-scoped
  strategy, research-run read/report, and deterministic eval smoke tools.
- Controlled draft/review/runtime MCP writes:
  narrow controlled tools wrap existing service-layer logic for draft
  artifacts, staged source notes, WorkerBundle validation/staging, draft
  review requests, checked counterexample evidence staging, non-accepted
  failure-log appends, research-run provenance, and strategy review exports.
- Service-layer boundary:
  MCP tools call typed Python services or CLI-equivalent policy surfaces. They
  do not expose arbitrary shell, environment inspection, hosted provider calls,
  accepted writes, promotion, human-review creation, or verifier-result
  mutation.
- Operator runbooks and demos:
  framework docs describe a CLI-first operator flow, MCP configuration as an
  optional adapter, workspace demo steps, final verification expectations, and
  PR summary requirements.
- Public KB policy smoke:
  public KB docs and guard checks explicitly reject treating operator, MCP,
  provider, LLM, checked-evidence, research-run, or strategy output as human
  review, verifier pass, accepted knowledge, or promotion authority.
- Optional Skill package:
  `docs/skills/cosheaf-operator/SKILL.md` documents how a Codex-like operator
  should use Cosheaf. It is a runbook, not runtime authority.

## Boundaries

- CLI remains the human and CI oracle.
- MCP is an optional local adapter. It is not the primary truth path and is
  not required for ordinary workspace use.
- MCP/operator output is not proof, checked evidence, verifier evidence,
  verifier pass, gate pass, human review, accepted status, accepted refutation,
  or promotion authority.
- Controlled writes may create only draft, proposal, review-context, or
  runtime records already allowed by Cosheaf policy.
- MCP does not write accepted KB content, promote artifacts, mark human
  review, mutate verifier results, call hosted providers, expose arbitrary
  shell, or bypass validation, gates, review, or promotion policy.
- Provider real-run behavior remains explicit, consented, policy-scoped, and
  default-off. CI and default tests do not call real hosted providers or
  require API keys.
- Accepted knowledge still requires validation, gates, human review where
  policy requires it, verifier evidence where applicable, source metadata for
  accepted public artifacts, and explicit promotion.
- Skipped verifier, provider, SAT, SMT, Lean, lake, optional-tool, network, or
  operator results are not passes.
- A successful Lean `#check` means only import and symbol resolution. It does
  not prove informal/formal semantic alignment.
- Public KB accepted content still requires source metadata and human review.

## Known Limitations

- No hosted provider call is default-on.
- No real provider call runs in CI or default tests.
- MCP remains optional and local. It is an adapter over whitelisted Cosheaf
  operations, not a new authority layer.
- SAT, SMT, plain Lean, and external Lean reference checking remain minimal
  optional invocation paths.
- External Lean library reference checking does not fetch CSLib/mathlib or
  prove semantic alignment.
- Strategy plans are useful planning guidance, not mathematical truth.
- Checked evidence is useful review context, not accepted mathematical truth.
- Research-run records are useful provenance, not proof or review authority.
- External public KB integration remains through local workspace roots, not a
  hosted registry service.
