# v0.11.0 - External Operator Campaigns

Status: published. Package metadata and `cosheaf.__version__` record
`0.11.0`. The public tag and GitHub release are published, post-tag release
smoke passed, and downstream workspace-template/public-KB pins are aligned to
`@v0.11.0`.

This release packages the V16 External AI Operator Harness + Bounded Multi-Run
Campaigns line on top of the published `v0.10.0` Cross-Check Evidence +
Checker Registry baseline.

## Included Surfaces

- Durable campaign sidecars under `.cosheaf/campaigns/<campaign-id>/`:
  `campaign.json`, attempts, operator results, scorecard, events, and scan
  report.
- Campaign lifecycle CLI:
  `cosheaf campaign start/show/append-attempt/scorecard/finalize`.
- External operator packet protocol v2:
  `cosheaf campaign next/export-task/import-result`.
- Deterministic campaign controller:
  `cosheaf campaign pause/resume/scan/run`.
- Campaign review handoff:
  `cosheaf campaign handoff <campaign-id> --out <dir> --json`.
- Deterministic campaign eval:
  `cosheaf eval campaign --json`.
- Framework ecosystem smoke coverage for campaign eval.
- Downstream workspace-template `make campaign-demo` coverage.
- Downstream public KB campaign-output policy guard coverage.

## Authority Boundary

Campaign records, attempts, scorecards, scans, operator packets, handoffs, and
campaign eval reports are review context only. They are not proof, source
metadata, human review, verifier pass, gate pass, accepted status, accepted
theorem/refutation, or promotion authority.

Skipped, unsupported, unavailable, and inconclusive rows are not passes.

This release does not add automatic theorem proving, Lean autoformalization,
CSLib/mathlib proof checking, informal/formal semantic alignment, automatic
accepted promotion, accepted writes, human-review creation, default hosted
provider calls, shell-backed campaign execution, web UI, production autonomy,
or multi-user permissions.
