# v0.4.0 - Strategy Planner + Research Task Graph

Status: release candidate. Package metadata and `cosheaf.__version__` record
`0.4.0`. The public `v0.4.0` tag and GitHub release are not published yet.
Post-tag release smoke and downstream workspace/public-KB pin alignment remain
publication-closeout tasks.

This release candidate packages the Strategy Planner + Research Task Graph
work that landed after the published `v0.3.0` Checked Evidence + Research Run
Loop release. It turns checked-evidence, failure-memory, artifact, issue,
context-pack, and research-run provenance into deterministic planning guidance
for Codex-style external operators while preserving the existing validation,
gate, human review, public/private, verifier, provider, MCP, and promotion
boundaries.

This release candidate is not production-ready software. It does not add
automatic theorem proving, full Lean/mathlib/CSLib proof checking of informal
statements, automatic autoformalization, automatic informal/formal semantic
alignment, automatic accepted promotion, AI-as-human-review, a web UI, SaaS
behavior, multi-user permissions, default-on provider calls, or provider/MCP
authority expansion.

## Highlights

- Strategy task graph model:
  `StrategyPlan`, `StrategyProblem`, `StrategyTaskGraph`,
  `StrategyTaskNode`, and `StrategyNextStep` records provide strict Pydantic
  DTOs for bounded planning guidance.
- Public schemas:
  `schemas/research_strategy.schema.json` and
  `schemas/research_task_graph.schema.json` document the persisted strategy
  and task-graph shapes.
- Deterministic strategy CLI:
  `cosheaf strategy plan/show/graph/next` emits deterministic JSON plans and
  ranked next steps. Runtime plans stay under `.cosheaf/strategy/`.
- Research-run integration:
  `cosheaf strategy update-from-run` attaches completed, failed, and skipped
  run provenance to task nodes without turning it into proof, gate pass, human
  review, accepted status, or promotion authority.
- Context and readiness surfacing:
  context packs and retrieval audit output can summarize related strategy
  plans, while promotion readiness may show open strategy blockers as advisory
  warnings only.
- Review export:
  `cosheaf strategy export-review` writes explicit strategy review context
  only under `reviews/strategy/` and supports dry-run output.
- Deterministic eval and security coverage:
  `cosheaf eval strategy-planner --json` checks planning boundaries including
  failed-direction handling, candidate-vs-checked evidence labels,
  skipped-not-pass behavior, public/private leakage, and no accepted-write
  authority.
- Downstream demo and policy:
  workspace-template includes a strategy demo, public KB includes
  strategy-plan policy docs, and the framework ecosystem matrix includes
  strategy-planner rows.

## Boundaries

- Strategy plans are guidance for review and execution planning. They are not
  proof, checked evidence, verifier evidence, verifier pass, gate pass, human
  review, accepted status, accepted refutation, or promotion authority.
- Strategy review exports remain review context only.
- Failed and skipped research-run commands remain failed/skipped, not pass.
- Candidate counterexamples remain candidate labels until separate checked
  evidence or review supports a stronger status.
- Checked counterexample evidence remains evidence for review. It is not human
  review, accepted refutation, accepted status, or promotion authorization.
- Accepted knowledge still requires validation, gates, human review where
  policy requires it, verifier evidence where applicable, source metadata for
  accepted public artifacts, and explicit promotion.
- Skipped verifier, provider, SAT, SMT, Lean, lake, optional-tool, network, or
  operator results are not passes.
- A successful Lean `#check` means only import and symbol resolution. It does
  not prove informal/formal semantic alignment.
- Provider real-run behavior remains explicit and default-off.
- MCP remains optional adapter work and is not the primary release path.
- Public KB accepted content still requires source metadata and human review.

and private workspace repositories into one mixed tree.

## Known Limitations

- No hosted provider call is default-on.
- No real provider call runs in CI or default tests.
- SAT, SMT, plain Lean, and external Lean reference checking remain minimal
  optional invocation paths.
- External Lean library reference checking does not fetch CSLib/mathlib or
  prove semantic alignment.
- Strategy plans are useful planning guidance, not mathematical truth.
- Checked evidence is useful review context, not accepted mathematical truth.
- Research-run records are useful provenance, not proof or review authority.
- External public KB integration remains through local workspace roots, not a
  hosted registry service.
