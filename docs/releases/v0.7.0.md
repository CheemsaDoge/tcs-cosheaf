# v0.7.0 - Bounded Research Loop + Attempt Memory

Status: published. Package metadata and `cosheaf.__version__` record `0.7.0`.
The public tag, GitHub release, post-tag release smoke, and downstream pin
alignment are complete.

This release packages the Bounded Research Loop + Attempt Memory work
landed after the published `v0.6.0` Operator Session + Review Handoff
release. It adds bounded multi-attempt research loops with deterministic
next-action planning, external operator task/result protocol, runtime attempt
memory, repeat-failure avoidance, retry-justification enforcement, loop
scanning, and deterministic eval coverage.

This release is not production-ready software. It does not add production
autonomy, automatic theorem proving, full Lean/mathlib/CSLib proof checking of
informal statements, automatic autoformalization, automatic informal/formal
semantic alignment, automatic accepted promotion, AI-as-human-review, a web UI,
SaaS behavior, multi-user permissions, default-on hosted provider calls, or
operator authority over accepted knowledge. Research-loop records, attempt
memory, task packets, and operator results are review context only: they are
not proof, verifier pass, gate pass, human review, source metadata, accepted
status, accepted refutation, or promotion authority.

## Highlights

- Bounded research loop records:
  strict runtime DTOs and deterministic storage under ignored
  `.cosheaf/research-loops/` paths record issue-scoped loops with budget,
  stop conditions, attempts, and review summaries.
- Core loop CLI:
  `cosheaf research-loop start/show/list/append-attempt/finalize` manages
  the lifecycle of a bounded research loop without calling hosted providers,
  writing accepted KB, creating human review, or promoting artifacts.
- Deterministic runner and operator protocol:
  `cosheaf research-loop next/step/run --dry-run/export-task/import-result`
  provides deterministic next-action planning, external operator task packet
  export, and operator result import. `run` supports dry-run planning only;
  non-dry-run execution is explicitly refused until a later deterministic
  implementation.
- Runtime attempt memory:
  deterministic `.cosheaf/research-loops/attempt-memory.json` index records
  attempt metadata, clusters failures, and surfaces cross-loop repeat-failure
  warnings in `next` and task packets.
- Retry-justification guard:
  `import-result` refuses repeated failed directions unless
  `retry_justification` is present, preventing unchallenged retries of known
  dead paths.
- Loop scanner:
  `cosheaf research-loop scan <loop-id> --json` checks loop runtime JSON,
  attempt JSON, and event logs for secrets, provider payloads, hidden reasoning,
  public-only private paths, accepted-write references, and authority claims.
- Deterministic research-loop eval:
  `cosheaf eval research-loop --json` runs fixture-based deterministic
  evaluation covering loop validity, attempt schema validity, repeat-failure
  detection, retry-justification blocks, public/private leaks, scanner accuracy,
  policy overclaim rejection, budget-stop accuracy, skipped-not-pass accounting,
  and accepted-write violations.
- Ecosystem matrix coverage:
  framework research-loop eval and workflow smoke rows, downstream
  workspace-template research-loop demo coverage, and public-KB research-loop
  policy guard alignment are integrated into the no-network three-repository
  ecosystem matrix.

## Limitations

- The GitHub release is published at
  <https://github.com/CheemsaDoge/tcs-cosheaf/releases/tag/v0.7.0>.
- The annotated tag object is
  `02cacce9c417ba3ba342fca0777af2580fc33a48`, and it peels to main commit
  `5a49833114251b7d59b53a7b17222239fb299f9d`.
- Post-tag install smoke from `@v0.7.0` passed.
- Non-dry-run `run` is explicitly refused; only dry-run planning is supported.
- Loop handoff export beyond explicit operator task JSON packets is not yet
  implemented.
- Research-loop success never means accepted status. Loop records are review
  context only.
- Formal links in public KB artifacts remain metadata unless a real checker
  verifies them.
- Skipped verifier results are not passes.
- Validation/gate success is not a substitute for human review of accepted
  public KB artifacts.
- This release does not add automatic theorem proving, Lean semantic alignment,
  hosted-provider defaults, production autonomy, or accepted-promotion authority.
