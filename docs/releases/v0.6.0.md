# v0.6.0 - Operator Session + Review Handoff

Status: release candidate. Package metadata and `cosheaf.__version__` record
`0.6.0`.

This release-candidate task does not create the public `v0.6.0` tag, publish a
GitHub release, run post-tag release smoke, or update downstream
workspace-template/public-KB pins. Those publication steps remain the separate
release closeout task after the release-candidate PR is reviewed and merged.

This release candidate packages the Operator Session + Review Handoff work that
landed after the published `v0.5.0` Operator MCP + Codex Application Layer
release. It adds a replayable, redacted, review-context handoff layer over the
existing CLI/MCP/service surfaces while preserving Cosheaf's validation, gate,
human review, public/private, verifier, provider, and promotion boundaries.

This release candidate is not production-ready software. It does not add
automatic theorem proving, full Lean/mathlib/CSLib proof checking of informal
statements, automatic autoformalization, automatic informal/formal semantic
alignment, automatic accepted promotion, AI-as-human-review, a web UI, SaaS
behavior, multi-user permissions, default-on provider calls, or operator-session
authority over accepted knowledge.

## Highlights

- Operator session records:
  strict runtime DTOs and deterministic storage under ignored
  `.cosheaf/operator-sessions/` paths record bounded metadata for an
  issue-scoped operator session.
- Operator session CLI:
  `cosheaf operator session start/show/append-check/append-ref/finalize/scan`
  records safe check and reference metadata without executing arbitrary
  commands, writing accepted KB content, creating human review, or promoting
  artifacts.
- Optional MCP session recording:
  whitelisted MCP tool calls can attach bounded metadata to an operator session
  when a caller supplies a session ID. MCP remains optional and usable without
  session recording.
- Leak scanning:
  session scan checks for secrets, provider dumps, hidden reasoning markers,
  private/public leakage, accepted-write attempts, and authority overclaims.
- Review handoff bundles:
  finalized sessions can build compact runtime handoff bundles that summarize
  checks, references, scanner findings, known limitations, and human review
  follow-up items.
- Explicit review-context export:
  `cosheaf operator handoff export --handoff <handoff-id> --dry-run --json`
  previews export targets, and non-dry-run export writes only review-context
  YAML under `reviews/operator/`.
- Downstream smoke:
  workspace-template operator-session demo coverage and public KB operator
  handoff policy checks are integrated into the ecosystem matrix.

## Boundaries

- CLI remains the human and CI oracle.
- MCP remains an optional local adapter, not the primary truth path.
- Operator sessions, MCP recordings, leak scans, handoff bundles, and handoff
  exports are review context only.
- Operator-session and handoff output is not proof, checked evidence, verifier
  evidence, verifier pass, gate pass, human review, source metadata, accepted
  status, accepted refutation, or promotion authority.
- Handoff export writes only explicit review-context YAML under
  `reviews/operator/`.
- Sessions, MCP, handoff bundles, and exports do not write accepted KB content,
  promote artifacts, mark human review, mutate verifier results, call hosted
  providers by default, expose arbitrary shell, or bypass validation, gates,
  review, or promotion policy.
- Provider real-run behavior remains explicit, consented, policy-scoped, and
  default-off. CI and default tests do not call real hosted providers or
  require API keys.
- Accepted knowledge still requires validation, gates, human review where
  policy requires it, verifier evidence where applicable, source metadata for
  accepted public artifacts, and explicit promotion.
- Skipped verifier, provider, SAT, SMT, Lean, lake, optional-tool, network,
  MCP, session, or eval results are not passes.
- A successful Lean `#check` means only import and symbol resolution. It does
  not prove informal/formal semantic alignment.
- Public KB accepted content still requires source metadata and human review.

## Known Limitations

- No hosted provider call is default-on.
- No real provider call runs in CI or default tests.
- MCP remains optional and local. It is an adapter over whitelisted Cosheaf
  operations, not a new authority layer.
- Operator sessions and handoff bundles are review context, not proof or
  review authority.
- Handoff export produces review-context YAML only; it does not create human
  review or accepted knowledge.
- SAT, SMT, plain Lean, and external Lean reference checking remain minimal
  optional invocation paths.
- External Lean library reference checking does not fetch CSLib/mathlib or
  prove semantic alignment.
- Strategy plans are useful planning guidance, not mathematical truth.
- Checked evidence is useful review context, not accepted mathematical truth.
- Research-run records are useful provenance, not proof or review authority.
- External public KB integration remains through local workspace roots, not a
  hosted registry service.
