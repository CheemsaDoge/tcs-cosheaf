# v0.10.0 - Cross-Check Evidence + Checker Registry

Status: release candidate. Package metadata and `cosheaf.__version__` record
`0.10.0`.

The public `v0.10.0` tag and GitHub release are not published yet. Post-tag
release smoke and downstream workspace/public-KB pin updates remain Phase F.2
work after this release-candidate PR merges.

This release candidate packages the V15 Cross-Check Evidence line on top of
the published `v0.9.0` Reviewable Research Workflow MVP baseline.

## Included Surfaces

- Typed checker registry:
  `cosheaf checker list`, `cosheaf checker describe`,
  `cosheaf checker run`, and `cosheaf checker run-suite`.
- Normalized checker statuses:
  `pass`, `fail`, `error`, `skipped`, `inconclusive`, `unsupported`, and
  `blocked_by_policy`.
- Runtime checker-run sidecars under `.cosheaf/checker-runs/<run-id>/`.
- Workflow cross-check reports:
  `cosheaf workflow cross-check <workflow-id> --json`.
- Workflow evidence reports:
  `cosheaf workflow evidence-report <workflow-id> --json`.
- Cross-check export:
  `cosheaf workflow export-crosscheck <workflow-id> --out <path> --json`.
- Gap taxonomy and exports:
  `cosheaf gap list <workflow-id> --json` and
  `cosheaf gap export <workflow-id> --out <path> --json`.
- Deterministic checker/cross-check eval:
  `cosheaf eval checker-crosscheck --json`.
- Ecosystem smoke coverage for the framework checker/cross-check eval row.

Downstream V15 closeout has also landed before this RC:

- workspace-template PR #85 adds `make crosscheck-demo` and
  `docs/CROSSCHECK_DEMO.md`;
- public KB PR #99 extends the repository-local policy guard so cross-check,
  evidence, gap, checker sidecar, and checker/cross-check eval reports remain
  non-authoritative public review context only.

## Authority Boundary

Checker registry output, checker-run sidecars, cross-check reports, evidence
reports, gap reports, and checker/cross-check eval output are review context
only. They are not proof, source metadata, human review, verifier pass without
a real checker result, gate pass, accepted status, accepted theorem/refutation,
or promotion authority.

Skipped, unsupported, unavailable, and inconclusive checker results are not
passes. Optional SAT, SMT, Lean, and lake tooling remain optional. Missing
optional tools must report skipped/unsupported rather than crash the core
workflow or count as pass.

This release candidate does not add automatic theorem proving, Lean
autoformalization, CSLib/mathlib proof checking, informal/formal semantic
alignment, automatic accepted promotion, accepted writes, human-review
creation, default hosted-provider calls, web UI, production autonomy, or
multi-user permissions.

## Release-Candidate Verification

The release-candidate PR must run:

```bash
python -m cosheaf.cli version --json
make lint
make typecheck
make test
make validate
make gate
python scripts/ecosystem_smoke.py --matrix --cosheaf "python -m cosheaf.cli" --framework-root . --workspace-template-root ../tcs-cosheaf-workspace-template --public-kb-root ../tcs-kb-public --json
git diff --check
```

Rows that require network access, a future tag, or unavailable optional tools
must remain skipped, not pass. Tag publication, GitHub release publication,
post-tag release smoke from `@v0.10.0`, and downstream pin updates are not part
of this release-candidate PR.
