# v0.9.0 - Reviewable Research Workflow MVP + Benchmark Harness

Status: published release. Package metadata and `cosheaf.__version__` record
`0.9.0`.

The public `v0.9.0` tag and GitHub release are published at:

```text
https://github.com/CheemsaDoge/tcs-cosheaf/releases/tag/v0.9.0
```

This release packages the deterministic execution-kernel baseline from
`v0.8.0` and adds the first reviewable-workflow CLI surface:

```bash
cosheaf workflow start --issue <issue-id> --query <query> --json
cosheaf workflow step <workflow-id> --json
cosheaf workflow readiness <workflow-id> --json
```

The current workflow implementation is intentionally conservative and thin.
`start` emits a workflow JSON record with an authority notice. `step` and
`readiness` are currently ephemeral CLI surfaces and do not yet persist or load
the planned `.cosheaf/workflows/<workflow-id>/` runtime state. Draft proposal
generation, workflow handoff build/scan/export, and
`cosheaf eval reviewable-workflow --json` remain follow-up implementation
work.

Workflow records, loop/session/operator records, draft proposals, and handoffs
remain review context only. This release does not add accepted writes, human
review creation, verifier-pass authority, gate-pass authority, source metadata
authority, accepted-refutation authority, production autonomy, default hosted
provider calls, automatic theorem proving, Lean semantic alignment, or
promotion authority.
